# 用户需求追踪板块更新指南

## 🎯 功能说明

`update_user_feedback.py` 脚本会自动从 `data/user_feedback.json` 读取数据，智能更新 HTML 报告中的"用户需求追踪"板块，并**自动将最新日期设为默认 active**。

## ✨ 核心特性

1. **自动识别最新日期**：从 JSON 数据中解析日期并排序，确保最新日期在最前面
2. **智能 active 状态管理**：只有最新日期的按钮和内容块带有 `active` 类
3. **日期格式统一**：自动转换为 "M 月 D 日" 格式（如 4 月 1 日）
4. **去重排序**：按日期降序排列，避免重复

## 📋 使用步骤

### 1. 添加新数据到 JSON

在 `data/user_feedback.json` 中添加新的日期数据：

```json
[
  {
    "date": "2026 年 4 月 7 日",
    "channels": [
      {
        "name": "微博",
        "content": "<h4>用户关注点分布</h4>..."
      },
      {
        "name": "小红书",
        "content": "<ul><li>热点话题...</li></ul>"
      }
    ]
  },
  // ... 其他日期数据
]
```

### 2. 运行更新脚本

```bash
cd /Users/jiewen/.real/users/user-777da3823a68e71779b041c8e7807df0/workspace/wangzhe_report

```

### 3. 推送更新到 GitHub Pages

```bash

git push origin main
```

### 4. 等待 GitHub Pages 刷新

通常 3-5 分钟后刷新网站即可看到最新内容。

## 📝 注意事项

1. **JSON 数据格式**：确保 `user_feedback.json` 符合规范，包含 `date` 和 `channels` 字段
2. **日期格式**：使用 "YYYY 年 M 月 D 日" 格式（如 "2026 年 4 月 7 日"）
3. **HTML 文件**：脚本仅更新 `index_with_tabs.html`，不影响 `index.html`
4. **备份建议**：重大更新前可备份 HTML 文件

## 🛠️ 与旧脚本的区别
